## [5.6.8-beta] — 2026-03-14

### Добавлено
- **Вкладка «Flatpak» — экспорт и импорт данных**: две плавающие кнопки «Экспорт» и «Импорт» в правом нижнем углу вкладки. Экспорт позволяет выбрать что сохранить (список приложений, репозитории, переопределения прав, данные `~/.var/app/` без папок cache) и папку назначения; данные приложений синхронизируются инкрементально через `rsync` — при повторном экспорте идентичные файлы пропускаются, что значительно ускоряет процесс. После успешного завершения создаётся файл `backup.ok` с датой. Импорт проверяет наличие маркера — если он отсутствует, предупреждает о возможной неполноте бэкапа. При выборе данных приложений показывается предупреждение о длительном времени первого экспорта.
- **Вкладка «Flatpak» — шифрование бэкапа (AES-256)**: в диалоге экспорта добавлен чекбокс «Зашифровать бэкап (AES-256)» с двумя полями пароля (с подтверждением). При включении шифрования бэкап создаётся потоковым pipe `tar | gpg` без промежуточных temp-файлов — единственный файл `flatpak-backup.gpg` в выбранной папке; overhead шифрования минимален благодаря AES-NI. Импорт автоматически определяет наличие зашифрованного файла, запрашивает пароль и при ошибке выводит понятное сообщение; расшифровка идёт в temp-директорию, которая удаляется после восстановления. Пароль передаётся через pipe (не видно в списке процессов).
- **Скрытие блока обновлений**: повторный клик на иконку «Обновления» в сайдбаре закрывает уже открытый блок проверки обновлений на странице «Начало»

### Добавлено
- **Вкладка «Flatpak» — проверка Flathub**: при открытии вкладки, если `flatpak` установлен, но репозиторий Flathub не подключён, всплывает диалог с предложением подключить его; при подтверждении устанавливает `flatpak-repo-flathub` и обновляет список приложений

### Изменено
- **Вкладка «Обслуживание» — общий прогресс**: группа «Различные баги и фиксы» исключена из счётчика и кнопки «Запустить все задачи»; фикс-задачи остаются доступны для ручного запуска

### Исправлено
- Кнопки «Установить» и «Установить бета» в блоке проверки обновлений: добавлен CSS-класс `pill` — скругления теперь соответствуют остальным кнопкам страницы «Начало»

## [5.6.8-beta] — 2026-03-12

### Изменено
- **Боковое меню навигации**: `ViewSwitcher` из хедера заменён на `Gtk.Paned` с боковой панелью навигации (`navigation-sidebar`); каждая вкладка — строка с иконкой и подписью; ширина панели регулируется перетаскиванием разделителя и сохраняется между запусками
- **Адаптивный sidebar**: при сужении панели ниже порога подписи автоматически скрываются (только иконки), при расширении — возвращаются; пункт «Текст у вкладок» из меню настроек удалён — поведение теперь полностью управляется шириной панели
- **Утилиты в нижней части sidebar**: кнопки «Обновления» и «Настройки» вынесены в нижнюю секцию боковой панели, отделены разделителем от основного меню; меню настроек открывается через `Gtk.PopoverMenu` без лишних кнопочных обёрток
- **Иконка настроек** `sliders-vertical-symbolic`: кастомная SVG-иконка с двумя регуляторами для пункта «Настройки» в sidebar; поддерживает `currentColor` для корректного отображения в светлой и тёмной теме
- **Парящий переключатель вкладок** (промежуточный этап): `ViewSwitcher` временно переносился в плавающую пилюлю (`Gtk.Overlay`) внизу окна; заменён боковой панелью навигации
- **Блок «Flatpak и Flathub» перенесён на вкладку «Flatpak»**: установка Flathub, уборка мусора, лечение и выдача доступа к /home теперь находятся на вкладке «Flatpak» вместо «Обслуживания»; счётчик прогресса на вкладке «Обслуживание» пересчитан без flatpak-задач

### Добавлено
- **Режим отладки `--debug`**: запуск `altbooster --debug` (или `python3 src/main.py --debug`) выводит в терминал все необработанные исключения Python с полным traceback, а также включает GLib/GTK-сообщения (`G_MESSAGES_DEBUG=all`, `GTK_DEBUG=interactive`)
- **Adw.Clamp для вкладок AMD и Intel**: `Adw.Clamp` добавлен в `DynamicPage` (базовый класс) и `IntelPage`, контент страниц AMD Radeon и Intel теперь также центрируется и ограничивается по ширине
- **Adw.Clamp для всех вкладок**: контент страниц теперь ограничен по ширине до 800px и центрируется на широких экранах (`Adw.Clamp` в `make_scrolled_page()`); при ширине окна < 600px clamp плавно сжимается вместе с окном — соответствует GNOME HIG
- **Выбор зеркала для обновлений ALT Linux**: перед запуском обновления системы открывается диалог выбора зеркала репозитория; выбранное зеркало применяется через `epm` перед выполнением `epm upgrade`
- **Предпросмотр пакетов перед обновлением**: перед обновлением системы отображается диалоговое окно со списком пакетов, которые будут установлены/обновлены; обновление запускается только после подтверждения пользователем
- **Вкладка «Flatpak»**: новый менеджер установленных Flatpak-приложений:
  - Список всех установленных приложений с иконками из appstream-кэша, версией и типом установки
  - Группировка по типу установки: «Пользовательские» и «Системные» (`Adw.ExpanderRow`)
  - Кнопка «Обновить» для каждого приложения отдельно (`flatpak update -y --user/--system`)
  - Кнопка «Удалить» с диалогом подтверждения (`flatpak uninstall -y --user/--system`)
  - Кнопка-снежинка ❄ для заморозки обновлений (`flatpak mask --user/--system`): нажата + синий акцент = заморожено; состояние перманентно хранится в конфигурации Flatpak
  - Кнопка «Обновить все» в заголовке (сначала user, затем system цепочкой)
  - Кнопка «Обновить список» для перезагрузки данных
  - Иконки приложений загружаются из appstream-каталогов (`~/.local/share/flatpak/appstream/` и `/var/lib/flatpak/appstream/`), масштабируются до 32×32 через GdkPixbuf; при отсутствии — generic-заглушка
  - Корректная обработка состояний: загрузка (спиннер), нет приложений, flatpak не установлен
  - User-операции выполняются без sudo; system-операции — через `backend.run_privileged()`


- **Вкладка Intel — scx_meteor**: новый раздел для управления кастомным CPU-планировщиком на базе sched_ext:
  - Проверка совместимости: наличие `/sys/kernel/sched_ext` в ядре и модель процессора
  - Установка scx_meteor из исходников (`git clone` + `cargo build --release`) с автоматической установкой зависимостей (rust, cargo, git, clang)
  - Создание systemd-сервиса `scx_meteor.service` с `ConditionPathExists=/sys/kernel/sched_ext`
  - Включение/выключение автозапуска (`systemctl enable/disable --now`)
  - Удаление бинарника и service-файла одной кнопкой
  - Красный плавающий баннер «Экспериментальная вкладка» с прозрачностью 60%
  - Иконка вкладки `processor-symbolic`, принудительно загружается из Adwaita независимо от активной темы (Papirus и т.д.)
- **DaVinci Resolve — Fairlight Audio**: активация записи звука в Fairlight и Edit одной кнопкой:
  - Устанавливает `alsa-plugins-pulse`, создаёт `/etc/asound.conf` и `~/.asoundrc` с маршрутом `pcm.!default { type pulse }` — обход бага pipewire-alsa (capture stream не стартует, `start_threshold=LONG_MAX`)
  - Кнопка `i` рядом со строкой — инструкция по настройке Fairlight после установки (Auto Patching, папка Capture)
  - После успешной установки автоматически открывается диалог с двумя шагами настройки
  - Кнопка «Применить снова» всегда активна (конфиг можно переприменить), выделена красным с прозрачностью 60%
- **Адаптивная раскладка (GNOME HIG)**: два режима в зависимости от ширины окна:
  - ≥ 620sp — нормальный режим (иконки + текст под ними)
  - < 620sp — только иконки с тултипом при наведении
  - Реализовано через `Adw.ToolbarView` + `Adw.BreakpointBin` + `Adw.Breakpoint`
- **Клавиатурные сокращения (GNOME HIG)**: `Ctrl+Q` — выход из приложения; `F1` — открыть диалог «О приложении»
- **Локализация .desktop**: добавлены `Name[ru]`, `GenericName[ru]`, `Comment[ru]`, `Keywords[ru]`; `Comment=` переведён на английский (требование freedesktop); категория `GTK;GNOME;` для корректной группировки в каталоге приложений
- **Переключатель «Текст у вкладок»** в гамбургер-меню: включает/выключает подписи под иконками вкладок; выбор сохраняется в конфиг пользователя; по умолчанию — только иконки (компактный режим)
- **Обрезка длинного названия пресета** в заголовке: название обрезается до ~9 символов с `…`; полное название доступно во всплывающей подсказке

### Исправлено
- **Иконки вкладок «Приложения» и «Intel» не отображались** (показывался символ потерянного файла): загрузка bundled-иконок переработана — теперь применяется тройная защита:
  1. `Gtk.IconTheme.add_search_path(icons/)` — GTK сразу видит `hicolor/index.theme` без перезапуска
  2. Удаление устаревшего `~/.local/share/icons/hicolor/icon-theme.cache`, который блокировал поиск новых иконок
  3. `invalidate_caches()` — принудительный сброс внутренних кэшей темы перед первой отрисовкой
- **SVG-иконки `grid-large-symbolic` и `processor-symbolic`** исправлены: hardcoded цвет заменён на `fill="currentColor"` — теперь иконки корректно адаптируются к светлой и тёмной теме

- **Обнаружение уже установленного Papirus на преднастроенных системах**: при нулевом запуске фоновый поток теперь дополнительно проверяет gsettings (`org.gnome.desktop.interface icon-theme`), чтобы определить, применена ли тема уже. Оба факта — установка пакета (`app_papirus_icons`) и активная тема (`papirus_applied`) — записываются в конфиг пользователя. Кнопка теперь показывает три корректных состояния: «Установить» / «Применить» / «Применено». На повторных запусках UI инициализируется из кэша state.json сразу, без ожидания фонового потока.

### Изменено
- **Адаптивная раскладка упрощена**: нижний `ViewSwitcherBar` удалён — все вкладки остаются в заголовке; минимальная ширина окна зафиксирована на 750px, при которой все вкладки гарантированно видны
- **Кнопки вкладок** автоматически уменьшаются при скрытии текста: `padding` сжимается до `4px 2px`, `margin` обнуляется — иконки располагаются компактнее

### Исправлено
- **Описания расширений** в списке установленных теперь отображаются на русском языке (вместо английского из `metadata.json`) для расширений из рекомендованного списка
- **UUID Pigeon** исправлен: `pigeon@subz69.gmail.com` → `pigeon@subz69.github` (фактический UUID установленного расширения)
- **Установка расширений при наличии системных пакетов с float-версией** (например, `gnome-shell-extension-simple-tiling`): `gext` падал с ошибкой pydantic (`version: Input should be a valid string, input_value=7.6`), блокируя установку любых расширений. Теперь при обнаружении таких расширений сначала предпринимается попытка исправить `metadata.json` через `sudo -n` (без промпта пароля); если прав нет — `gext` пропускается и используется нативный метод установки через API extensions.gnome.org
- **Threading в расширениях**: `win.stop_progress()` при установке и удалении системных расширений теперь корректно вызывается через `GLib.idle_add()` вместо прямого вызова из фонового потока
